# マーケティング自動化パターン

出典: https://note.com/tsukky739/n/n40f4ec2b506a (tsukky, 2026-03-12)

## 1. 定型データ収集の自動化

### 構成
- **MCP**: AIが外部サービス（広告媒体・スプレッドシート・Notion・Gmail等）に直接アクセス
- **gog CLI**: Google Sheets/Gmail/Calendarをコマンドラインから操作
- **Claude Code**: データ取得→集計→分析→提案を一気通貫

### ワークフロー
1. 「昨日の実績を教えて」→ AIがMCP経由で広告媒体からデータ取得
2. CPA・CVR等のKPIを自動計算
3. 仮説とネクストアクションを提案
4. 「スプレッドシートに記載して」→ gog CLI経由で自動転記
5. cron設定で毎朝自動実行も可能

### 適用例（マーケ以外）
- 営業: 受注データ → 日報自動生成
- 採用: 応募数確認 → 週次レポート
- 経理: 入出金チェック → 仕訳データ整理

## 2. データ分析のAI委託

### できること
- **コホート分析**: 月別新規ユーザーのN月後残存率
- **LTV試算**: チャネル・獲得経路別にリピート率×単価で算出
- **クリエイティブ効果比較**: 同期間の広告別CPA/CVR比較
- **予算配分最適化**: ROI実績→追加予算の配分提案

### 前提条件
- AIとデータが接続されていること（MCP設定済み）
- データの正確性（ゴミデータでは分析も無意味）
- 「AIの使い方」と同じくらい「AIにデータを繋ぐ設計」に投資が必要

## 3. CLAUDE.md設計の原則（非エンジニア向け）

### 自律動作の原則
```
・判断して進む（確認を最小化する）
  - 小さな技術的判断は自分で決めて実行
  - 「どうしますか？」は方向性不明時と破壊的操作前だけ
・長く動く（途中で止まらない）
  - エラー時は代替手段を自分で考えて試す
  - タスク完了後にまとめて報告
```

### 解説ルール（非エンジニアに有効）
```
エンジニア的な作業を行った場合、作業後にその場で
非エンジニア向けに「何をしたか・なぜそうしたか・仕組み」を
平易に解説すること。比喩や具体例を使って分かりやすく。
```

### MEMORY.md構成
- アカウントID対応表（広告媒体等）
- ユーザー操作権限（確認不要な操作の明示）
- MCP接続サービス一覧

## 4. クリエイティブ制作

### ツール
- **Nanobanana Pro** (Geminiベース): 既存素材→リサイズ・加工で8-9割完成
- **Sora2**: テキスト→動画生成（10本中2-3本使える）
- **Claude in Chrome**: ブラウザ上のCanva等を直接操作
