# 日本のビジネス書類 レイアウトルール

## 基本原則：位置 = 意味

横書きビジネス書類では、**位置がセマンティクスを決定する**。
ラベルだけでは意味が確定しない。

## 横書き書類の基本レイアウト

```
┌──────────────────────────────────────────────────┐
│ 【左上エリア】              【右上エリア】        │
│ ＝ 宛先・提出先             ＝ 差出人・発信者     │
│                                                  │
│ 例：                        例：                 │
│ ・○○工事作業所 御中        ・株式会社○○建設    │
│ ・工事名                    ・代表取締役○○      │
│ ・元請の事業所名            ・下請の会社名        │
├──────────────────────────────────────────────────┤
│ 【中央上部】                                     │
│ ＝ 書類タイトル・日付                            │
│                                                  │
│ 例：作業員名簿 / 令和○年○月○日                  │
├──────────────────────────────────────────────────┤
│ 【本文エリア】                                   │
│ ＝ 書類の内容                                    │
└──────────────────────────────────────────────────┘
```

## 適用される書類

| 書類 | 左上（宛先） | 右上（差出人） |
|------|-------------|---------------|
| 注文書 | 発注先（受注者） | 発注者（注文主） |
| 見積書 | 見積依頼者 | 見積提出者 |
| 請求書 | 請求先 | 請求者 |
| 作業員名簿 | 元請事業所・工事名 | 下請業者名 |
| 施工体制台帳 | 工事名・発注者 | 元請業者 |
| 再下請負通知書 | 元請・上位下請 | 再下請業者 |

## よくある記入ミス

### 作業員名簿の「事業所の名称」欄

```
【誤】左上の「事業所の名称」に下請業者名を記入
【正】左上の「事業所の名称」には元請の工事名・事業所名を記入
```

**理由**: 「事業所」というラベルだけ見ると自社のことと誤解しやすいが、
左上 = 宛先 なので、提出先である元請の情報を書く。

### 判定ロジック

```typescript
// セル位置から意味を判定
function inferSemanticsByPosition(cell: string, maxCol: number): 'recipient' | 'sender' | 'content' {
  const col = cellToColIndex(cell); // A=0, B=1, ...
  const row = cellToRowIndex(cell); // 1-indexed

  if (row <= 5) {
    // 上部エリア
    if (col < maxCol / 2) {
      return 'recipient'; // 左上 = 宛先
    } else {
      return 'sender';    // 右上 = 差出人
    }
  }
  return 'content'; // 本文
}
```

## AIへの指示例

```
事業所名を探す際は、以下のルールに従うこと：
1. 「事業所の名称」ラベルを探す
2. そのラベルが左上エリア（列がシート幅の半分より左）にある場合
   → これは「提出先事業所」= 元請の工事名
3. 右上エリアにある場合
   → これは「差出人事業所」= 下請業者名

左上の事業所名に下請業者名が入っていたらエラー。
```
