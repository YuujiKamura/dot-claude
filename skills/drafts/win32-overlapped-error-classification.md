## 1. ERROR_OPERATION_ABORTED vs Broken Pipe

Distinguishing local I/O cancellation from remote peer disconnection in Windows Named Pipes

### Events

1. AI misclassified 995 (ERROR_OPERATION_ABORTED) as a broken pipe/remote disconnect, leading to incorrect subscriber removal.
2. 995 (ERROR_OPERATION_ABORTED) occurs when a local handle issues CancelIoEx or closes; it is a completion status for a canceled operation, not a peer-driven disconnect.
3. Peer disconnection manifests as 109 (ERROR_BROKEN_PIPE) or 232 (ERROR_NO_DATA) on ReadFile/WriteFile.
4. Classification error caused the server to drop persistent subscribers when a local timeout-driven CancelIoEx occurred.

### Examples

```zig
// Correct classification in pipe_server.zig
const err = k32.GetLastError();
switch (err) {
 .OPERATION_ABORTED => return error.OperationAborted, // Self-cancel
 .BROKEN_PIPE, .NO_DATA => return error.BrokenPipe, // Remote disconnect
 else => return error.Unexpected,
}
```
