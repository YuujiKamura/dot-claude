# Known Issues Registry

レビューで発見された再発防止パターンをドメイン別に蓄積する。

## ファイル命名規則
- `<domain>.md` (例: `zig-com.md`, `rust-pdf.md`, `web-frontend.md`)

## エントリ形式
```markdown
## <パターン名>
- **検出条件:** <grepで引っかかるパターン or コード構造>
- **問題:** <何が起きるか>
- **正しい対処:** <どうすべきか>
- **発見日:** <YYYY-MM-DD>
- **発生元:** <どのレビューで見つかったか>
```

## 蓄積基準
- 同じ種類の問題が2回以上発生
- プロジェクト固有の落とし穴（一般的なミスではない）
- 検出可能な具体的パターンがある
